Introduction

Arduino is an open-source electronics platform based on easy-to-use hardware and software. It allows beginners, students, and professionals to quickly prototype and build interactive systems. Thanks to its simplicity and flexibility, Arduino has become one of the most popular tools for learning embedded systems and microcontroller programming.

Programming an Arduino is done using the Arduino IDE which is based on C/C++ with simplified functions to make development more accessible.

Basic structure of an arduino program

Every Arduino program, called a sketch, follows a simple structure composed of two main functions:

setup()

  • Runs once when the Arduino starts or resets.
  • Used to initialize pins, configure communication (like Serial), and set up sensors or modules.
void setup() { //configuration and initialization }

loop()

  • Runs continuously after setup() finishes.
  • Contains the main logic of your program and repeats indefinitely.
void loop() { //your main program }
Variables and constants

Variables are used to store data that your program can use and modify during execution. Constants store data that does not change. Choosing the right data type is important for memory efficiency and correct program behavior.

Variables

  • Declared with a data type followed by a name.
  • Can change value during program execution.

Constants

  • Use const (or #define) to define values that should not change.
  • Prevents accidental modifications and improves code readability.

Common data types

Type Size Description Example
int 2 bytes Whole numbers (-32,768 to 32,767) int counter = 0;
float 4 bytes Decimal numbers float temperature = 23.5;
boolean 1 byte True/False boolean ledOn = true;
char 1 byte Single character char letter = 'a';
String varies Text String name = "Arduino";
Input and Output

Arduino interacts with the outside world through its pins, which can be used as digital or analog inputs and outputs.

  • Input: Reading data from sensors or buttons to get information from the outside world
  • Output: Sending signals to actuators like LEDs, motors, or displays to control the environment.

Digital I/O

  • Digital pins can be either HIGH (1) or LOW (0)
  • Functions to use:
    • pinMode(pin, mode) configure pin as input or output
    • digitalWrite(pin, value) set output HIGH or LOW
    • digitalRead(pin) read input value

Analog I/O

  • Analog input: reads a voltage from 0V to 5V and converts it to a value between 0 and 1023.
    • analogRead(pin) returns integer value (0–1023)
  • Analog output (PWM): simulates analog voltage using Pulse Width Modulation.
    • analogWrite(pin) value between 0 (0%) and 255 (100%)

Note: Not every pin on an Arduino board can be used as an analog input or output. The number and location of analog pins depend on the specific Arduino model.

Function declarations

Functions allow you to organize your code, make it reusable, and improve readability. Instead of writing the same code multiple times, you can define a function once and call it whenever needed.

A function definition consists of:

  • The return type of the function (void if it does not return a value).
  • The name of the function. A list of parameters (inputs) enclosed in parentheses and separated by commas. The statements that define the function, enclosed in curly braces { }.
  • A list of parameters enclosed in parentheses and separated by commas.
  • The statements that define the function, enclosed in curly braces { }.

Example – Function that returns the square of a number:

int square(int number) { return number * number; }

Example – Function with no return value (void):

void blinkLED(int pin) { digitalWrite(pin, HIGH); delay(500); digitalWrite(pin, LOW); delay(500); }
Serial communication

Serial communication allows Arduino to send and receive data to and from a computer, which is essential for monitoring, debugging, and controlling your Arduino programs. The Serial Monitor in the Arduino IDE is used to view the data in real-time.

Initializing Serial Communication

Use Serial.begin(baudRate) in the setup() function to start communication at a specific speed (baud rate).

void setup() { Serial.begin(9600); // Start serial communication at 9600 bits per second }

Sending data

  • Serial.print() prints data without a newline
  • Serial.println() prints data and moves to a new line
void loop() { int sensorValue = analogRead(A0); Serial.print("Sensor value: "); Serial.println(sensorValue); delay(1000); // Wait 1 second }

Receiving data

  • Serial.available() checks if data is available to read
  • Serial.read() reads incoming data
  • Serial.readUntil(terminator) reads incoming serial data until a specific character(terminator) is detected
void loop() { if (Serial.available() > 0) { char incoming = Serial.read(); Serial.print("You sent: "); Serial.println(incoming); } }
Using libraries

Libraries in Arduino are pre-written code modules that simplify working with sensors, modules, displays, communication protocols, and more. They allow you to reuse code without writing everything from scratch

Including a library

#include <LibraryName.h>

Installing a library

  • Use Arduino IDE Library Manager: Sketch > Include Library > Manage Libraries…
  • Or manually place the library folder in the Arduino/libraries directory.
Good practices

Here are some recommendations for programming effectively on Arduino:

  • Use descriptive names for variables, functions, and constants.
  • Keep code modular by using functions for repeated tasks.
  • Avoid long delays; prefer millis() over delay() for non-blocking timing.
  • Comment your code to explain functionality and logic.
  • Initialize all variables to prevent undefined behavior.
  • Include only necessary libraries to save memory.
  • Check hardware limits (pin capabilities, voltage, current) before connecting devices.
